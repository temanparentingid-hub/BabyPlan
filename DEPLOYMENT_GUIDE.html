Panduan Deployment Cloudflare Stack

Langkah migrasi database ke Cloudflare D1 & KV dan deployment WebApp ke domain babyplan.my.id.

📋 Prasyarat Sebelum Memulai

  • Akun aktif di Cloudflare Dashboard.
  • Kepemilikan domain babyplan.my.id.
  • Aplikasi Node.js terinstal di komputer lokal Anda untuk menjalankan paket perintah.

🛠️ Langkah-Langkah Deployment

Langkah 0.A Membuat Repository GitHub Baru

Hubungkan folder lokal proyek Anda ke repository GitHub baru bernama `BabyPlan`:

# Jalankan di folder utama proyek (root BabyPlan)
git init
git add .
git commit -m "Initial commit - Cloudflare Stack setup"
git branch -M main
git remote add origin https://github.com/USERNAME_ANDA/BabyPlan.git
git push -u origin main
Langkah 0.B Konfigurasi DNS Domain babyplan.my.id ke Cloudflare

Arahkan pengelolaan DNS domain Anda ke Cloudflare agar Cloudflare Pages & Workers dapat dihubungkan:

  1. Masuk ke dashboard Cloudflare, klik Add a Site, lalu masukkan nama domain babyplan.my.id.
  2. Pilih paket gratis (Free plan) dan salin 2 alamat Nameservers yang disediakan.
  3. Masuk ke panel provider tempat Anda membeli domain babyplan.my.id, cari menu **Nameservers**, lalu ganti dengan Nameservers dari Cloudflare tersebut.
  4. Tunggu status domain menjadi **Active** di Cloudflare Dashboard (biasanya memakan waktu beberapa menit).
Langkah 1 Instalasi Perkakas Wrangler CLI

Buka terminal Anda, lalu arahkan ke direktori backend di dalam proyek Anda untuk menginstal module wrangler:

cd backend
npm install
Langkah 2 Hubungkan Terminal ke Cloudflare (Login)

Autentikasi akun Cloudflare Anda pada terminal lokal dengan menjalankan perintah:

npx wrangler login

Layar browser web Anda akan otomatis terbuka. Klik tombol Allow / Izinkan untuk menuntaskan login, kemudian kembali ke terminal Anda.

Langkah 3 Membuat Database D1 & KV Namespace

Buat instansi database SQL SQLite (D1) baru dan KV Storage untuk allowed_emails di edge network Cloudflare:

# 1. Membuat D1 Database
npx wrangler d1 create babyplan-db

# 2. Membuat KV Namespace
npx wrangler kv:namespace create BABYPLAN_KV
⚠️
Perhatikan Output ID! Setelah perintah selesai, catat string ID unik yang dihasilkan di layar terminal (misal: database_id dan id namespace). Anda membutuhkannya di langkah berikutnya.
Langkah 4 Sinkronisasi File wrangler.toml

Buka berkas konfigurasi Wrangler di backend/wrangler.toml pada editor kode Anda.

Perbarui ID penampung dengan ID riil yang Anda salin dari Langkah 3:

# Sesuaikan bagian KV & D1 dengan ID asli Anda
[[kv_namespaces]]
binding = "BABYPLAN_KV"
id = "MASUKKAN_ID_KV_NAMESPACE_DI_SINI"

[[d1_databases]]
binding = "DB"
database_name = "babyplan-db"
database_id = "MASUKKAN_ID_DATABASE_D1_DI_SINI"
Langkah 5 Terapkan Skema Tabel Database D1

Buat semua tabel relational database di Cloudflare menggunakan file skema SQL yang telah disediakan di backend/schema.sql:

npx wrangler d1 execute babyplan-db --remote --file=./schema.sql
Langkah 6 Deploy API Worker Backend & Binding Subdomain

Kirim kode router API Worker Anda yang tertulis di backend/src/index.js ke Cloudflare edge server:

npx wrangler deploy

Hubungkan Subdomain API Domain Anda:

  1. Masuk ke dashboard web Cloudflare.
  2. Masuk ke menu Workers & Pages > Pilih Worker **babyplan-backend**.
  3. Masuk ke tab Triggers > klik tombol **Add Custom Domain** di bawah kolom Custom Domains.
  4. Masukkan subdomain API Anda: api.babyplan.my.id, lalu simpan.
Langkah 7 Deploy Frontend Menggunakan Cloudflare Pages

Alih-alih Firebase Hosting, kita akan menggunakan **Cloudflare Pages** untuk meng-host frontend React secara serverless:

  1. Buka dashboard Cloudflare > masuk ke menu Workers & Pages > pilih tab Pages > klik **Create a project** > **Connect to Git**.
  2. Hubungkan akun GitHub Anda dan pilih repositori **`BabyPlan`**.
  3. Pada bagian **Build Settings**:
    • Framework Preset: Vite
    • Build command: npm run build
    • Build output directory: dist
  4. Buka bagian **Environment variables (advanced)**, tambahkan variabel baru:
    Key: VITE_API_URL
    Value: https://api.babyplan.my.id
  5. Klik **Save and Deploy**. Cloudflare Pages akan menarik kode Anda dari GitHub, melakukan kompilasi build, dan meng-host web secara otomatis.
  6. Setelah proses build selesai, buka proyek Pages tersebut di dashboard, pergi ke tab **Custom Domains**, klik **Set up a custom domain**, masukkan domain utama Anda: babyplan.my.id, lalu konfirmasi.
💡
Dengan metode Git Integration ini, setiap kali Anda melakukan git push origin main di masa mendatang, Cloudflare Pages akan otomatis mendeteksi perubahan dan memperbarui website Anda secara otomatis!
